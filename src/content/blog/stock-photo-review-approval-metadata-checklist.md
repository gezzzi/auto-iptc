# ストックフォト審査を通過するための画像メタデータチェックリスト

ストックフォトに写真を投稿する際、避けて通れないのが**審査プロセス**です。どんなに美しい写真でも、審査基準を満たしていなければ承認されません。

特に、**メタデータの不備**は審査落ちの主な原因の一つです。この記事では、主要ストックフォトサイトの審査基準と、承認率を上げるためのメタデータチェックリストを紹介します。

## ストックフォト審査の基本

### 審査で評価される項目

主要なストックフォトサイトでは、以下の項目が審査されます：

#### 1. 技術的品質
- 解像度
- ピント
- 露出
- ノイズレベル
- 色再現

#### 2. 商業的価値
- 需要のあるテーマか
- 汎用性があるか
- ユニークな視点か

#### 3. 法的要件
- モデルリリース（人物写真の場合）
- プロパティリリース（私有地・建物の場合）
- 著作権侵害の有無
- 商標・ブランドロゴの有無

#### 4. **メタデータの適切性**
- タイトルの正確性
- 説明文の詳細度
- キーワードの関連性
- 不適切なキーワードの有無

**メタデータの不備は、審査落ちの原因の約30〜40%を占めます。**

### 審査期間

| サイト | 審査期間 | 初回投稿 |
|--------|----------|----------|
| Adobe Stock | 1〜3日 | 7〜14日 |
| Shutterstock | 1〜2日 | 3〜7日 |
| PIXTA | 2〜5日 | 7〜10日 |
| iStock | 3〜7日 | 14〜30日 |

**初回投稿は審査が厳しく、時間もかかります。**

## メタデータに関する審査基準

### Adobe Stock

#### タイトル
- **要件:** 簡潔で正確
- **推奨長:** 6〜10語、60文字以内
- **NG例:**
  - 「Beautiful image」（抽象的すぎる）
  - 「IMG_1234.jpg」（ファイル名そのまま）
  - 「テスト」（意味不明）

#### 説明文
- **要件:** 詳細で使用シーンが明確
- **推奨長:** 1〜3文、150〜200文字
- **NG例:**
  - 「綺麗な写真です」（主観的）
  - 「F2.8, ISO400で撮影」（技術情報のみ）

#### キーワード
- **上限:** 50個
- **要件:** 関連性のあるキーワードのみ
- **NG例:**
  - 画像に写っていないもの（犬の写真に「猫」）
  - スパムキーワード（「最高」「完璧」「プロ」の乱用）
  - 競合サイト名（「Shutterstock」など）

### Shutterstock

#### タイトル
- **要件:** 具体的で直接的
- **推奨長:** 5〜10語
- **重視:** 商業利用のシーンが想像しやすいタイトル

#### 説明文
- **要件:** 商業的価値を明確に
- **推奨長:** 100〜150文字
- **重視:** どんな広告・記事に使えるかを記述

#### キーワード
- **上限:** 50個
- **要件:** 検索意図に合致するキーワード
- **重視:** 最初の7個のキーワードが特に重要

### PIXTA

#### タイトル
- **要件:** 日本語で自然な表現
- **推奨長:** 20〜40文字
- **重視:** 日本の文化・習慣に適した表現

#### 説明文
- **要件:** 丁寧な日本語
- **推奨長:** 50〜150文字
- **重視:** 使用シーンの具体例

#### キーワード
- **上限:** 50個
- **要件:** 日本語検索に適したキーワード
- **重視:** 日本特有の概念やイベント（お正月、桜、紅葉など）

## 審査落ちの主な原因

### メタデータ関連

#### 1. 不正確なキーワード

**❌NG例:**
```
画像: 猫の写真
キーワード: 犬, ライオン, ペット, 動物園
```

**理由:** 画像に写っていない「犬」「ライオン」「動物園」が含まれている

**✅OK例:**
```
画像: 猫の写真
キーワード: 猫, ペット, 動物, かわいい, 室内, 家, ネコ科
```

#### 2. キーワードスパム

**❌NG例:**
```
キーワード: 綺麗, 美しい, 素敵, 最高, 完璧, プロ, ベスト, 高品質, 一流, トップ
```

**理由:** 主観的・抽象的なキーワードの羅列

**✅OK例:**
```
キーワード: オフィス, ビジネス, 働く, 仕事, デスク, パソコン
```

#### 3. 不適切な説明文

**❌NG例:**
```
この写真は私が初めて撮影した作品です。とても気に入っています。
```

**理由:** 個人的な感想・背景は不要

**✅OK例:**
```
モダンなオフィスで働く若い女性。明るい自然光が差し込む空間で、ノートパソコンを使って集中して作業をしています。企業サイトやビジネス記事に最適です。
```

#### 4. コピーアンドペーストの使い回し

**❌NG例:**
```
写真A（オフィス）: ビジネスシーンの写真です。
写真B（自然風景）: ビジネスシーンの写真です。
写真C（食べ物）: ビジネスシーンの写真です。
```

**理由:** 明らかに間違っているメタデータ

**✅OK例:**
```
写真A（オフィス）: モダンなオフィスで働く女性
写真B（自然風景）: 富士山と桜の春景色
写真C（食べ物）: 新鮮な有機野菜の詰め合わせ
```

#### 5. 言語の不統一

**❌NG例:**
```
タイトル: Business woman working in office
説明文: オフィスで働く女性の写真
キーワード: ビジネス, office, 仕事, work, 企業, company
```

**理由:** 日本語と英語が混在

**✅OK例（日本語統一）:**
```
タイトル: オフィスで働く女性ビジネスマン
説明文: モダンなオフィスで働く若い女性...
キーワード: ビジネス, オフィス, 仕事, 企業...
```

**✅OK例（英語統一）:**
```
Title: Business woman working in office
Description: Young professional woman working...
Keywords: business, office, work, company...
```

## 審査通過のためのチェックリスト

### 投稿前の最終確認

#### ✅ 画像品質

□ 解像度が4MP以上ある
□ ピントが合っている
□ 適切な露出である
□ ノイズが少ない
□ 色が正確に再現されている
□ ブロックノイズがない（JPEG圧縮）

#### ✅ 法的要件

□ 人物が写っている場合、モデルリリースを取得している
□ 私有地・建物が写っている場合、プロパティリリースを取得している
□ ブランドロゴ・商標が写り込んでいない
□ 著作権のあるデザイン・作品が写っていない
□ 個人を特定できる情報（住所、車のナンバーなど）がない

#### ✅ タイトル

□ 簡潔で具体的（6〜10語、60文字以内）
□ 主要な被写体が含まれている
□ 自然な日本語（または英語）
□ ファイル名ではない
□ 主観的な表現を避けている（「綺麗」「素敵」など）

#### ✅ 説明文

□ 1〜3文、150〜200文字
□ 5W1H（誰が、何を、いつ、どこで、なぜ、どのように）が含まれている
□ 使用シーンや用途が明記されている
□ 客観的な表現
□ 個人的な感想・背景を含めていない

#### ✅ キーワード

□ 10〜30個のキーワードを設定（上限: 50個）
□ すべてのキーワードが画像に関連している
□ 画像に写っていないものを含めていない
□ 重要なキーワードを最初に配置
□ スパムキーワード（「最高」「完璧」など）を避けている
□ 競合サイト名を含めていない
□ 言語が統一されている（日本語のみ、または英語のみ）

#### ✅ フォーマット

□ JPEG形式である（写真の場合）
□ 色空間がsRGBである
□ ファイルサイズが適切（3〜10MB程度）
□ IPTCメタデータが埋め込まれている

#### ✅ リリース書類

□ 必要なリリースがすべてアップロードされている
□ リリース書類が鮮明である
□ 署名がある
□ 日付が記入されている

## 審査落ちした場合の対処法

### 1. 却下理由を確認

多くのサイトでは、却下理由が通知されます。

**主な却下理由:**
- 技術的品質不足（ピンボケ、ノイズなど）
- メタデータの不備
- 類似画像が多すぎる
- 商業的価値が低い
- 法的問題（リリース不足など）

### 2. 修正可能な場合は修正

**メタデータの問題の場合:**
- タイトル・説明文・キーワードを見直し
- 不適切なキーワードを削除
- より詳細な説明文に書き直し

**技術的問題の場合:**
- 編集で改善できる場合は再編集
- 改善できない場合は再撮影

### 3. 再投稿

修正後、再度投稿します。

**注意点:**
- 同じ問題で再度却下されないよう、十分に確認
- 別の写真を投稿し続けることも重要

### 4. 学習として活用

却下された写真から学び、次回の投稿に活かします。

**学習ポイント:**
- どんな点が問題だったか
- 似たような写真で改善できることは何か
- 今後避けるべきパターン
